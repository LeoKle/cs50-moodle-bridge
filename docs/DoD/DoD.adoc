= Definition of Done
:toc:
:toclevels: 2

== Purpose

This document defines when the application is considered *done* and *ready for release*.
It does not define requirements, but specifies the conditions under which implemented
requirements are accepted.

Example::
The requirement "Export grades to Moodle" is done only if grades are exported correctly
and verified in Moodle.

---

== Functional Requirements

All defined features are implemented and working as specified.

* Courses can be created
* Students can be imported
* CS50 and Moodle data can be uploaded
* Grading is calculated correctly
* Grades can be exported to Moodle
* A grading overview is available

---

== Correctness of Grading

The grading logic produces correct and consistent results.

* Passed and failed assignments match CS50 data
* Percentages are calculated and displayed correctly
* No missing grades exist

---

== Error Handling and Validation

The system responds to invalid input with error messages.

* Invalid files / files format are rejected
* Missing data does not crash the system
* Clear error messages are shown to the user

---

== Security

Sensitive data is protected.

* Only authorized users can export grades

---

== Performance

The application performs within acceptable limits.

* Imports and grading complete in reasonable time
* The user interface remains responsive

---

== Usability

The application is easy to understand and use.

* Main workflows are intuitive
* Error and success messages are clear
* No manual database interaction is required

---

== Testing

The application is sufficiently tested.

* Core functions (grading, import, export) are tested
* All tests pass before release
* No known critical bugs are open

---

== Documentation

Required documentation is available and up to date.

* arc42 documentation exists in the repository and is complete enough to understand the system
** User workflows are documented
* Guiding code lines exist in the repository andd are up to date
** Known limitations are listed

---

== Deployment and Build

The system can be deployed reliably.

* Application builds without errors
* Configuration is environment-independent
* Deployment steps are documented

---

== Stakeholder Acceptance

The system is approved by stakeholders.

* Tutor use cases are validated
* Feedback is addressed or documented
* Final approval is given
